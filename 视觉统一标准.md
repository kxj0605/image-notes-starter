# 数字生活空间｜视觉统一标准

> 本标准用于统一首页、任务、笔记、AI、日历、设置等页面的视觉与交互语言。后续新增页面和组件应优先复用本标准，不再为单个页面重新定义风格。

## 1. 设计定位

### 1.1 核心气质

产品整体应呈现以下感受：

- 温和、治愈、高级、专注
- 低刺激、低饱和、适合长期使用
- 像一个愿意每天打开的“数字生活空间”，而不是冷硬的效率工具

### 1.2 风格公式

> 灰蓝背景 + 白色大圆角卡片 + 轻玻璃感 + 奶油渐变点缀 + 超低饱和语义色

视觉语言可参考 Apple 的克制、Linear 的秩序、Craft 的柔和与 Notion 的信息组织，但不直接复制其中任何一种风格。

### 1.3 设计原则

1. **内容优先**：装饰服务于信息，不抢夺注意力。
2. **柔和克制**：减少高饱和色、重边框、纯黑和强烈阴影。
3. **系统一致**：相同层级使用相同颜色、圆角、间距和阴影。
4. **低密度呼吸感**：保留充足留白，不为填满页面而堆叠模块。
5. **状态有含义**：颜色用于表达功能和状态，而不是随意装饰。

## 2. 基础视觉 Token

### 2.1 色彩系统

#### 中性色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `color-bg-page` | `#F4F7FB` | 全站主背景 |
| `color-bg-card` | `#FFFFFF` | 普通卡片、弹窗、输入区域 |
| `color-bg-glass` | `rgba(255,255,255,.72)` | 玻璃卡片、悬浮导航 |
| `color-text-primary` | `#111827` | 标题、主要正文 |
| `color-text-secondary` | `#667085` | 说明、次要信息 |
| `color-text-muted` | `#98A2B3` | 时间、占位符、弱提示 |
| `color-line-soft` | `rgba(17,24,39,.07)` | 必须出现时的弱分隔线 |

使用规则：

- 页面背景统一使用灰蓝色，不在不同页面切换纯白、深灰或米黄背景。
- 正文避免纯黑，优先使用带蓝灰倾向的深色。
- 边框不是主要分层方式，优先通过留白、背景明度和轻阴影区分层级。

#### 语义色

| 状态 | 浅背景 | 前景色 | 用途 |
| --- | --- | --- | --- |
| 信息蓝 | `#EAF3FF` | `#3976D5` | 链接、信息、当前项 |
| 成功绿 | `#E8F7F0` | `#247B55` | 完成、成功、正常状态 |
| 提醒橙 | `#FFF2DE` | `#C97808` | 待处理、提醒 |
| 危险红 | `#FFE8EC` | `#D93655` | 删除、失败、高风险 |
| 创意紫 | `#F0EAFF` | `#7647C8` | AI、灵感、特殊能力 |

语义色应以浅背景、小图标、小标签和局部状态出现，禁止铺成大面积高饱和色块。绿色仅作为成功或完成状态色，不作为全站主题色。

### 2.2 奶油渐变

系统仅保留以下四组低饱和渐变：

```css
/* 蓝系 */
linear-gradient(135deg, #EAF3FF 0%, #F5F8FF 100%);

/* 紫系 */
linear-gradient(135deg, #F0EAFF 0%, #F8F4FF 100%);

/* 粉系 */
linear-gradient(135deg, #FFEAF3 0%, #FFF5FA 100%);

/* 青系 */
linear-gradient(135deg, #E8F7F5 0%, #F4FBFA 100%);
```

渐变只用于重点卡片、笔记封面、图标背景和空状态插画，不作为整页背景，也不同时在一个模块中使用过多色系。

### 2.3 间距系统

全站优先使用以下间距：

| Token | 数值 | 典型用途 |
| --- | ---: | --- |
| `space-1` | `8px` | 图标与文字、紧凑元素 |
| `space-2` | `16px` | 标题与说明、表单项内部 |
| `space-3` | `24px` | 卡片内边距、卡片间距 |
| `space-4` | `32px` | 页面模块间距 |
| `space-5` | `48px` | 大区块分隔、页面上下留白 |

允许在紧凑控件中使用 `4px` 或 `12px`，但不应随意产生 `13px`、`19px`、`27px` 等无系统数值。

### 2.4 圆角系统

| Token | 数值 | 使用对象 |
| --- | ---: | --- |
| `radius-sm` | `12px` | 按钮、标签、紧凑控件 |
| `radius-md` | `16px` | 输入框、下拉菜单 |
| `radius-lg` | `20px` | 小卡片、列表项 |
| `radius-xl` | `24px` | 主卡片、弹窗、浮层 |
| `radius-full` | `999px` | 胶囊标签、头像、开关 |

大面积容器优先使用 `24px`。同类组件必须使用相同圆角，不因页面不同而变化。

### 2.5 阴影与玻璃效果

```css
--shadow-sm: 0 3px 10px rgba(52, 63, 84, .05);
--shadow-md: 0 10px 28px rgba(52, 63, 84, .07);
--shadow-lg: 0 18px 42px rgba(52, 63, 84, .09);
```

- `shadow-sm`：按钮、标签、轻悬浮元素。
- `shadow-md`：普通卡片、下拉菜单。
- `shadow-lg`：弹窗、重点卡片、悬浮面板。
- 玻璃卡片建议使用 `backdrop-filter: blur(20px)`，并保留半透明白色背景。
- 禁止使用厚重黑影、发光描边和多个互相冲突的阴影方向。

### 2.6 字体与层级

字体优先使用：

```css
font-family: Inter, "PingFang SC", "Microsoft YaHei", system-ui, sans-serif;
```

| 层级 | 建议字号 | 字重 | 用途 |
| --- | ---: | ---: | --- |
| 页面标题 | `32px` | `700` | 每页唯一主标题 |
| 模块标题 | `20–24px` | `600` | 功能区标题 |
| 卡片标题 | `16–18px` | `600` | 卡片名称 |
| 正文 | `14–16px` | `400` | 主要内容 |
| 辅助文字 | `12–14px` | `400` | 时间、状态、说明 |

正文行高建议为 `1.6`，标题行高为 `1.25–1.35`。避免同时出现过多字号和字重。

## 3. 组件统一规则

### 3.1 卡片

标准卡片：

```css
background: #FFFFFF;
border-radius: 24px;
padding: 24px;
box-shadow: var(--shadow-md);
border: 1px solid rgba(17, 24, 39, .04);
```

- 边框仅用于提升浅色背景上的可辨识度，不形成明显轮廓。
- 卡片内部按“标题—说明—内容—操作”排列。
- 同一页面避免嵌套超过两层卡片。

### 3.2 按钮

- 主按钮：深蓝灰背景、白色文字，保持克制而明确。
- 次按钮：白色或半透明背景、深色文字、轻阴影或弱边框。
- 危险按钮：只在删除等高风险动作中使用红色。
- 默认高度建议 `40–44px`，圆角 `12px`，左右内边距 `16–20px`。
- 同一区域只保留一个视觉权重最高的主按钮。

### 3.3 输入框

- 白色或极浅灰背景，圆角 `16px`。
- 默认使用弱边框；聚焦时用信息蓝描边或柔和外环。
- 标签、提示和错误信息固定位置，避免聚焦后布局跳动。
- 不使用强烈内阴影模拟凹陷效果。

### 3.4 图标

- 统一使用线性图标，建议 `2px` 描边。
- 常规尺寸为 `18–20px`。
- 图标容器为 `32–40px` 的圆角矩形，使用低饱和浅色背景。
- 同一功能的图标在不同页面中必须保持一致。
- 禁止混用线性、面性、3D 和不同描边粗细的图标。

### 3.5 标签与状态

- 使用胶囊形或 `12px` 圆角的小标签。
- 背景使用语义色的浅色版本，文字使用对应深色版本。
- 标签表达状态、分类或权限，不作为无意义装饰。
- 单张卡片建议不超过 3 个彩色标签。

### 3.6 插画与视觉素材

统一采用“扁平渐变 + 柔和几何形状”方向。

- 颜色来自既定奶油渐变和语义色。
- 避免写实 3D、强拟物材质与扁平插画混用。
- 插画用于引导、空状态和情绪表达，不取代功能信息。

## 4. 页面结构规范

### 4.1 通用骨架

所有主页面优先采用同一信息节奏：

```text
页面标题与一句说明
↓ 32px
概览 / 核心操作
↓ 32px
主要内容区域
↓ 32px
次要内容 / 历史记录 / 辅助信息
```

页面左右留白和内容最大宽度保持一致；不要让某一页满屏铺开、另一页又过度收窄。

### 4.2 页面示例

| 页面 | 第一层 | 第二层 | 第三层 |
| --- | --- | --- | --- |
| 首页 | 今日概览 | 快捷入口 | 最近内容 |
| 任务页 | 任务概览 | 今日任务 | 长期任务 / 四象限 |
| 笔记页 | 笔记概览 | 最近笔记 | 全部笔记 |
| AI 页 | AI 助手 | 快捷功能 | 历史记录 |
| 日历页 | 日期概览 | 日历主体 | 当日事项 |
| 设置页 | 账户概览 | 分类设置 | 次要与危险操作 |

四象限任务区域应表现为可拖拽的任务面板，而不是静态图表；卡片移动后同步改变任务所属象限。

### 4.3 响应式原则

- 桌面端优先保留宽阔留白和多列卡片。
- 平板端减少列数，不压缩卡片内部间距。
- 移动端改为单列，页面边距建议 `16px`，卡片内边距可降至 `16–20px`。
- 触控目标不得小于 `40 × 40px`。

## 5. 动效与交互反馈

- 常规过渡时长：`160–240ms`。
- 悬停仅做轻微上浮、明度或阴影变化，位移建议不超过 `2px`。
- 页面切换与弹窗出现应柔和，不使用明显弹跳、旋转或炫技动画。
- 拖拽、保存、删除、加载必须有清晰状态反馈。
- 尊重系统的“减少动态效果”设置。

---

## 6. 禁止事项

- 不为单独页面创建新的主题色或背景色。
- 不把绿色重新扩展为全站主色。
- 不大面积使用高饱和色、纯黑或强对比渐变。
- 不混用多套圆角、阴影、图标和按钮风格。
- 不依赖粗边框划分所有区域。
- 不为了显得丰富而塞满卡片、标签和装饰图形。
- 不让不同页面拥有完全不同的信息骨架。

---

## 7. 开发落地建议

建议将基础 Token 集中为 CSS 变量，组件只引用变量，不重复写具体色值：

```css
:root {
  --color-bg-page: #F4F7FB;
  --color-bg-card: #FFFFFF;
  --color-bg-glass: rgba(255, 255, 255, .72);
  --color-text-primary: #111827;
  --color-text-secondary: #667085;
  --color-text-muted: #98A2B3;
  --color-line-soft: rgba(17, 24, 39, .07);

  --space-1: 8px;
  --space-2: 16px;
  --space-3: 24px;
  --space-4: 32px;
  --space-5: 48px;

  --radius-sm: 12px;
  --radius-md: 16px;
  --radius-lg: 20px;
  --radius-xl: 24px;
  --radius-full: 999px;

  --shadow-sm: 0 3px 10px rgba(52, 63, 84, .05);
  --shadow-md: 0 10px 28px rgba(52, 63, 84, .07);
  --shadow-lg: 0 18px 42px rgba(52, 63, 84, .09);
}
```

新增组件前应先检查是否已有同类组件；新增颜色、圆角或阴影前，应说明现有 Token 为什么无法满足需求。

---

## 8. 视觉验收清单

每个新页面交付前检查：

- [ ] 页面背景、内容宽度和左右留白与其他页面一致
- [ ] 颜色均来自中性色、语义色或四组奶油渐变
- [ ] 绿色只用于成功、完成等语义状态
- [ ] 间距主要来自 `8 / 16 / 24 / 32 / 48`
- [ ] 同类卡片、按钮、输入框使用统一圆角
- [ ] 阴影只使用 `sm / md / lg` 三档
- [ ] 图标风格、尺寸和描边一致
- [ ] 页面遵循“标题—概览—主要内容—次要内容”的骨架
- [ ] 彩色标签和装饰没有抢夺正文注意力
- [ ] 桌面、平板、移动端均保持可读和可操作
- [ ] 悬停、聚焦、加载、成功、错误状态完整
- [ ] 整体仍然呈现温和、治愈、高级、专注的产品气质
