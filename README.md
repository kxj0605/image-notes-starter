# 日程笔记

一个用于练习全栈网站开发的个人效率工具。它把笔记、任务、月历、四象限和仪表盘放在同一个私人工作台中，并允许用户公开分享部分笔记。

> 项目仍在迭代中。当前功能以实际代码为准，产品规划见《网站修改需求文档.md》。

## 已实现功能

- 邮箱注册、登录和退出。
- 私人工作台：今日任务、长期任务、最近笔记和今日重点提醒。
- 笔记：新建、编辑、删除，以及私密 / 公开设置。
- 任务：新建、编辑、删除、完成、恢复和四种进展状态。
- 任务视图：列表、月历、四象限。
- 任务筛选：按进展状态和重要紧急程度筛选。
- 公开笔记：未登录用户可浏览，登录用户可发表评论。
- 评论：作者可编辑或删除自己的评论。
- 个人设置：修改昵称，切换默认或薄荷工作台主题。
- 未登录首页使用锁定预览，不展示看起来像真实用户数据的私人内容。

## 当前未实现

- 任务表格视图。
- 四象限拖拽。当前可在任务编辑区修改所属象限。
- 笔记图片附件、搜索、标签、提醒和分享链接。
- 团队协作与复杂项目管理。

## 技术栈

- React 19
- Vite 7
- Supabase（登录、数据库）
- Lucide React
- 原生 CSS
- GitHub + Vercel

## 本地运行

需要先安装 Node.js，然后在项目根目录执行：

```bash
npm install
npm run dev
```

构建生产版本：

```bash
npm run build
```

## 环境变量

在项目根目录创建 `.env.local`：

```env
VITE_SUPABASE_URL=你的 Supabase 项目地址
VITE_SUPABASE_PUBLISHABLE_KEY=你的 Supabase Publishable Key
```

不要把 `.env.local`、真实地址或密钥提交到 Git。

## Supabase 数据

前端当前使用以下数据表：

- `profiles`：用户昵称等资料。
- `notes`：笔记内容与可见性。
- `tasks`：任务、日期、状态和四象限分类。
- `comments`：公开笔记评论。

评论表及其权限策略可参考 `supabase-comments.sql`。基础表结构和 RLS 策略需要在对应 Supabase 项目中正确配置；只修改前端不能替代数据库权限控制。

## 项目结构

```text
src/
├─ main.jsx              # 工作台、笔记、任务、公开笔记和评论
├─ styles.css            # 全站样式
├─ config.js             # 页面、任务状态和四象限配置
├─ supabaseClient.js     # Supabase 初始化
├─ pages/
│  ├─ HomePage.jsx       # 首页与锁定预览
│  └─ AuthPages.jsx      # 登录与注册
└─ utils/
   ├─ date.js            # 日期工具
   └─ tasks.js           # 任务筛选与排序工具
```

## 产品与视觉约束

- 产品定位是轻量的个人记录与任务管理工具，不是团队协作平台。
- 私密笔记和个人任务只能由所属用户访问；公开区域显示昵称，不显示完整邮箱。
- 默认视觉为浅灰蓝背景、白色或半透明卡片和低饱和语义色；绿色主要用于成功、完成等状态。
- 更完整的产品边界见《网站修改需求文档.md》，视觉规则见《视觉统一标准.md》。
